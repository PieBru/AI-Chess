Open source Runs fully offline No install, no account

A chess engine lab that fits in one HTML file

Granpy Piero's AI Chess is a browser chess app where you can play a friend, a hand-written engine you can actually read, Stockfish at full strength, or an LLM of your choosing — and watch how differently each one thinks.

That little bar isn't decoration. Every AI move gets scored against a local search, so you can watch — move by move — which engine actually played the stronger game.

The pitch

Most "chess AI" demos on the web are one engine, hidden behind a server you can't inspect, that you can't run without an internet connection. Granpy Piero's AI Chess takes the opposite approach: it's a single .html file — open it and you have a full chess rules engine, a hand-built search-based AI with five difficulty levels, a hook into Stockfish (currently the strongest chess engine that exists), and an option to let any OpenAI-compatible LLM take the wheel. No build step, no server, no npm install. Save the file, double-click it, and you're playing.

Four very different opponents

The interesting part isn't any single opponent — it's that you can put them in the same room and watch them differ.

Tier 1

A friend, across the same screen

Pass-and-play, full rules support — castling, en passant, underpromotion, the works.

Tier 2 — Normal AI

An engine you can actually read

Alpha-beta search, iterative deepening, quiescence search, a tapered evaluation function — five difficulty levels from "loses on purpose" to "the strongest this hand-built engine can play." Every line of it is sitting right there in the file.

Tier 3 — Grandmaster

Stockfish, at full strength

Runs entirely client-side via WebAssembly in a Web Worker, so it never freezes the board. This is the same engine class that outclasses every human who has ever played, Kasparov included.

Tier 4 — LLM-Assisted

Bring your own model

Point either side at any OpenAI-compatible chat endpoint — a hosted model, or a local one on your own machine — and watch it play. It gets the legal moves handed to it directly, so it can't hallucinate an illegal one; it just has to pick well.

"Fully local" is not a slogan here

Play Human vs. Human, Human vs. Normal AI, or even Human vs. Grandmaster and the app never touches the network — the entire rules engine, the search algorithm, and Stockfish itself all run in your browser. Stockfish's WebAssembly build ships bundled right next to the single HTML file, so Grandmaster is fully offline too: no CDN fetch, no internet required. LLM mode is opt-in by definition — you supply the endpoint.

Local-first doesn't mean network-hostile. It means the network is optional, not load-bearing.

Under the hood, briefly

For the curious: the board is a classic 0x88 array, the kind of representation you'll find in small engines going back decades. Move generation is fully legal (not just pseudo-legal-and-hope), verified against the standard perft test suite at multiple depths, including a stress position that exercises castling, en passant, and promotions together in one go.

$ perft startpos depth 4 got 197281, expected 197281 → OK $ perft kiwipete depth 3 got 97862, expected 97862 → OK

The Normal engine's search is negamax with alpha-beta pruning, a transposition table, and quiescence search so it doesn't blunder a piece right past its own search horizon. Grandmaster mode runs the single-threaded Stockfish WASM build specifically so it works without special server headers — no cross-origin isolation dance required, just open the file.

Why bother comparing them at all?

Because "how good is this AI, really?" is usually answered with a marketing number, not something you can watch happen. Every AI move — Normal, Grandmaster, or LLM — gets graded against a quick local search and tagged: best, good, an inaccuracy, a mistake, a blunder. Set two AIs against each other and you get a genuinely legible answer to "which one is actually smarter," move by move, instead of a leaderboard score you have to take on faith.
